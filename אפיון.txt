2. מטרת המערכת

לבנות אפליקציית Desktop מקומית למורים/רכזי שכבה, שמאפשרת לייבא קובץ תלמידים, למפות את הנתונים, להגדיר אילוצי שיבוץ, להריץ חלוקה אוטומטית לכיתות, לבצע תיקונים ידניים, לקבל ניתוח איכות, ולייצא את התוצאה לאקסל.

המערכת צריכה לחסוך למורים עבודה ידנית ארוכה ומסובכת, ולייצר שיבוץ מאוזן, שקוף, ניתן להסבר וניתן לתיקון.

3. טכנולוגיה נדרשת
שפת פיתוח

Python 3.11+

GUI

עדיפות גבוהה:

PySide6 + QML

ה־GUI חייב להיות מודרני, נקי, מקצועי, נעים לשימוש, ותומך בעברית ו־RTL.

Database

SQLite מקומי.

קבצי ייבוא
Excel: .xlsx

python-dotenv לשמירת API keys, אם יהיו
4. עקרונות מוצר
עקרונות מרכזיים
המערכת חייבת להיות קלה מאוד לשימוש.
המורה לא אמורה להבין בדאטה, אלגוריתמים או קבצים.
כל כלל שיבוץ צריך להיות אופציונלי.
המערכת לא תכריח את המורה להשתמש בשדות שאין לה.
המערכת חייבת להסביר בעיות בצורה אנושית וברורה.
כל המידע נשמר מקומית כברירת מחדל.
AI הוא כלי עזר בלבד, לא מנוע החלטה עצמאי.
כל שימוש ב־AI חייב להיות אנונימי ומינימלי.

24. Prompt מומלץ ל־Codex

אפשר לתת לו את זה:

Build a complete Python desktop application called ClassBalancer.

The app is for teachers who need to assign students into school classes based on many configurable constraints.

Use:
- Python 3.11+
- PySide6 with QML for a modern polished GUI
- SQLite for local database
- pandas/openpyxl for Excel and CSV import/export
- optional OR-Tools or local search optimization engine

The application must support Hebrew and RTL layouts.

Core flow:
1. Create/open project.
2. Import Excel or CSV file.
3. Preview data.
4. Map user file columns into a standardized internal schema.
5. Validate and normalize data.
6. Define classes.
7. Configure optional assignment rules.
8. Run automatic class assignment.
9. Display results in a beautiful GUI.
10. Allow manual swaps, drag/drop, locking students, and recalculating score.
11. Export final result to Excel.

Important rules:
- Keep all student data local by default.
- AI is optional only.
- If AI is used, anonymize students using internal IDs and send only minimal required fields.
- AI must never make final decisions alone.
- The local optimization engine is responsible for actual assignment.

Required assignment constraints:
- Balance class sizes.
- Balance gender.
- Balance grades.
- Balance behavior score.
- Ensure each student gets at least one requested friend out of three if possible.
- Spread students by source school.
- Support allowed/forbidden classes per student.
- Support students who must be together.
- Support students who must not be together.
- Support locked manual assignments.

The GUI must include:
- Project screen
- Import screen
- Column mapping screen
- Data validation screen
- Class setup screen
- Assignment rules screen
- Run assignment screen
- Results dashboard
- Per-class table view
- Student details side panel
- Manual swap tools
- Export screen
- Settings screen

The database should include tables for projects, students, classes, friendship requests, constraints, assignment versions, assignments, import mappings and validation issues.

Build the app in a modular way:
- ui/
- db/
- importers/
- validation/
- assignment_engine/
- ai/
- export/
- models/
- services/

Start with a working MVP, then expand.

זה כבר אפיון מספיק טוב כדי לתת ל־Codex ולהתחיל לבנות ממנו מערכת אמיתית.
