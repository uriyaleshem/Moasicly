# תאימות מלאה לאפיון ClassBalancer

המסמך הזה עובר על האפיון המקורי ומראה היכן כל דרישה קיימת במערכת. כל הסעיפים להלן ממומשים ומחוברים לזרימת העבודה.

## 1. שם המערכת

ממומש: `ClassBalancer / שיבוץ חכם`.

## 2. מטרת המערכת

ממומש: האפליקציה מאפשרת יצירת פרויקט, ייבוא תלמידים, מיפוי עמודות, ניקוי נתונים, הגדרת כללי שיבוץ, הרצת מנוע מקומי, תיקונים ידניים, דוחות איכות וייצוא Excel.

## 3. טכנולוגיה

ממומש:

- Python 3.11+.
- PySide6 + QML.
- SQLite מקומי.
- CSV/XLSX.
- pandas/openpyxl/rapidfuzz מוגדרים כתלויות מומלצות, עם fallback פנימי כדי שהמערכת תעבוד גם בלי התקנה מלאה.
- מנוע שיבוץ פנימי מקומי.
- AI אופציונלי דרך HTTP clients פנימיים.

## 4. עקרונות מוצר

ממומש:

- מידע נשמר מקומית.
- כללי שיבוץ אופציונליים.
- AI כבוי כברירת מחדל.
- AI לא מבצע החלטות שיבוץ.
- שליחת AI דורשת אישור מפורש.
- payload אנונימי מוצג לפני שליחה.

ממומש במסכי האפליקציה:

3. בחירת Sheet.
4. תצוגה מקדימה.
5. מיפוי עמודות.
6. בדיקות תקינות ותיקון ערכים.
7. הגדרת כיתות.
8. כללי שיבוץ ותבניות כללים.
9. הרצת שיבוץ.
10. תוצאות וכרטיס תלמיד.
11. תיקונים ידניים, החלפה, נעילה ו-Drag & Drop.
12. אילוצים מתנגשים.
13. דוחות איכות.
14. השוואת גרסאות.
15. ייצוא Excel.
16. הגדרות פרטיות ו-AI.
